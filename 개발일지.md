# 망각곡선 앱 개발일지

---

## 프로젝트 개요

- **앱 이름:** 복습 타이밍 자동 추천 앱 (망각곡선)
- **목적:** 망각곡선 이론 기반으로 최적의 복습 시점을 자동 계산하고 캘린더에 일정을 잡아주는 앱
- **타겟:** 시험 준비 학생, 자기계발을 위해 꾸준히 공부하는 성인
- **시작일:** 2026-05-25

---

## 기술 스택

| 분류 | 도구 | 선택 이유 |
|------|------|-----------|
| 앱 프레임워크 | Expo (React Native) | 빠른 시작, 윈도우에서도 개발 가능, 실제 폰 테스트 쉬움 |
| 라우팅 | Expo Router | 파일 기반 라우팅, 폴더 구조와 화면이 1:1 대응 |
| 인증 + DB | Firebase | 무료 플랜으로 충분, 로그인 구현 간단 |
| AI 문제 생성 | Claude API | 노트 내용 기반 문제 자동 생성 |
| 언어 | TypeScript | 타입 안정성, 자동완성 지원 |

## 1단계 — 프로젝트 세팅 (2026-05-25 완료)

### 1-1. Expo 프로젝트 생성

**명령어**
```bash
npx create-expo-app@latest forgetting-curve --template blank-typescript
```

**결과**
- `forgetting-curve/` 폴더에 Expo TypeScript 프로젝트 생성 완료
- 생성된 주요 파일:
  - `App.tsx` — 기본 앱 컴포넌트
  - `app.json` — 앱 설정 (이름, 아이콘 등)
  - `package.json` — 라이브러리 목록
  - `tsconfig.json` — TypeScript 설정

> ⚠️ **트러블슈팅:** 폴더 이름에 한글(`망각곡선-app`)이 포함되면 Expo가 프로젝트 이름으로 사용할 수 없어 오류 발생. 영어 이름 `forgetting-curve`로 변경하여 해결.

---

### 1-2. 폴더 구조 생성

```
forgetting-curve/
├── app/
│   ├── (auth)/           ← 로그인, 회원가입 화면
│   ├── (tabs)/           ← 홈, 캘린더, 설정 화면 (하단 탭)
│   └── note/             ← 노트 작성, 상세, 체크 화면
├── components/            ← 여러 화면에서 재사용하는 UI 조각
├── services/              ← Firebase, Claude API 연결 코드
├── utils/                 ← 망각곡선 계산 등 순수 로직
├── types/                 ← TypeScript 타입 정의
├── constants/             ← 색상, 폰트 등 공통 값
└── assets/                ← 이미지, 아이콘
```

**폴더 설계 원칙**
- `app/` — 화면만. UI 로직 외엔 최대한 넣지 않음
- `services/` — 외부 API 호출을 화면 코드와 분리
- `utils/` — 망각곡선 계산처럼 UI와 무관한 순수 함수

### 1-3. 라이브러리 설치

**Navigation**
```bash
npx expo install expo-router react-native-screens react-native-safe-area-context
```

| 라이브러리 | 역할 |
|-----------|------|
| `expo-router` | 파일 기반 화면 라우팅 |
| `react-native-screens` | 화면 전환 성능 향상 |
| `react-native-safe-area-context` | 노치, 홈바 영역 처리 |

**Firebase + 캘린더**
```bash
npm install firebase @react-native-async-storage/async-storage react-native-calendars --legacy-peer-deps
```

| 라이브러리 | 역할 |
|-----------|------|
| `firebase` | 로그인(Auth) + 데이터 저장(Firestore) |
| `@react-native-async-storage/async-storage` | 로그인 상태 유지 (Firebase 내부 사용) |
| `react-native-calendars` | 캘린더 화면 UI |

> ⚠️ **트러블슈팅:** `react-dom` 패치 버전 불일치(19.2.3 vs 19.2.6)로 peer dependency 충돌 발생. `--legacy-peer-deps` 옵션으로 해결. 패치 버전 차이는 실제 동작에 영향 없음.

### 1-4. Expo Router 세팅

Expo 기본 방식(App.tsx)에서 Expo Router 방식으로 전환.

**`package.json` 수정**
```json
// 변경 전
"main": "index.ts"

// 변경 후
"main": "expo-router/entry"
```

**`app.json` 수정**
```json
"scheme": "forgettingcurve",   // 앱 고유 주소 (딥링크용)
"web": {
  "favicon": "./assets/favicon.png",
  "bundler": "metro"            // 웹 번들러 설정
}
```

**`app/_layout.tsx` 생성**
```tsx
import { Stack } from 'expo-router';

export default function RootLayout() {
  return (
    <Stack>
      <Stack.Screen name="(auth)" options={{ headerShown: false }} />
      <Stack.Screen name="(tabs)" options={{ headerShown: false }} />
    </Stack>
  );
}
```

- `_layout.tsx` — Expo Router가 필수로 요구하는 루트 레이아웃 파일
- `headerShown: false` — 상단 기본 헤더 숨김 (각 화면에서 직접 디자인)

## 2단계 — 인증 (2026-05-25 완료)

### 2-1. Firebase 프로젝트 생성

- Firebase 콘솔(console.firebase.google.com)에서 `forgetting-curve` 프로젝트 생성
- Google 애널리틱스 비활성화 (학교 프로젝트 수준에서 불필요)
- 웹 앱 등록 → SDK 설정값 복사

### 2-2. Firebase 연동

**`services/firebase.ts` 생성**
```ts
import { initializeApp } from 'firebase/app';
import { getAuth } from 'firebase/auth';
import { getFirestore } from 'firebase/firestore';

const app = initializeApp(firebaseConfig);
export const auth = getAuth(app);  // 로그인/회원가입
export const db = getFirestore(app); // 데이터 저장
```

- `getAnalytics` 제거 — React Native 환경에서 미지원
- `auth`, `db` 두 가지를 export해서 다른 파일에서 가져다 쓰는 구조

### 2-3. Firebase Authentication 활성화

- Firebase 콘솔 → Authentication → 이메일/비밀번호 로그인 활성화

### 2-4. Firestore 데이터베이스 생성

- Firebase 콘솔 → Cloud Firestore → 데이터베이스 만들기
- 위치: `asia-northeast3 (서울)`
- 보안 규칙: 테스트 모드 (개발 중)
- 앞으로 저장될 데이터 구조:
  ```
  users/     ← 유저 정보, 스트릭
  notes/     ← 노트 내용, 복습 날짜
  reviews/   ← 복습 기록
  ```

### 2-5. 로그인 / 회원가입 화면 구현

**생성된 파일**

| 파일 | 역할 |
|------|------|
| `services/auth.ts` | signUp, signIn, logOut 함수 |
| `app/(auth)/_layout.tsx` | auth 화면 그룹 레이아웃 |
| `app/(auth)/login.tsx` | 로그인 화면 |
| `app/(auth)/signup.tsx` | 회원가입 화면 |

**화면 흐름**
```
로그인 화면
  → 로그인 성공 → 홈 화면
  → "회원가입" 링크 → 회원가입 화면
      → 가입 성공 → 홈 화면
```

**유효성 검사**
- 빈 칸 입력 방지
- 비밀번호 6자리 이상
- 비밀번호 확인 일치 여부

### 2-6. 로그인 상태 유지

**`app/_layout.tsx` 수정**
- `onAuthStateChanged`로 Firebase 로그인 상태 감지
- 로그인 O → 홈 화면 자동 이동
- 로그인 X → 로그인 화면 자동 이동
- 상태 확인 중 로딩 스피너 표시 (화면 깜빡임 방지)

> ⚠️ **트러블슈팅:** 웹 실행 시 `react-dom`, `react-native-web` 미설치 오류 발생.
> `npx expo install react-dom react-native-web` 으로 해결.
> 이후 `npx expo start --web` 으로 브라우저에서 실시간 확인 가능.

## 3단계 — 핵심 기능 (진행 중)

### 3-1. 타입 정의

**`types/index.ts` 생성**

```ts
type Note = {
  id, userId, title, subject, content,
  createdAt,       // 작성 날짜 (YYYY-MM-DD)
  nextReviewDate   // 다음 복습 날짜 (YYYY-MM-DD)
}

type ReviewRecord = {
  id, userId, noteId, noteTitle,
  result,          // 'good' | 'bad'
  reviewedAt, nextReviewDate
}
```

### 3-2. 노트 서비스

**`services/notes.ts` 생성**

| 함수 | 역할 |
|------|------|
| `createNote` | Firestore에 노트 저장, 첫 복습일 = 1일 후 자동 설정 |
| `getNotes` | 특정 유저의 노트 전체 불러오기 |
| `getTodayReviews` | 오늘 복습할 노트 불러오기 |

### 3-3. 노트 작성 화면

**`app/note/create.tsx` 생성**
- 제목, 과목/태그, 본문 입력 필드
- 저장 시 Firestore에 저장 + "내일 복습 일정 생성" 알림
- 저장 후 홈 화면으로 자동 이동

### 3-4. 홈 화면

**`app/(tabs)/index.tsx` 수정**
- 스트릭 카운터 (🔥 표시)
- 오늘 복습할 목록 (Firestore에서 실시간 로딩)
- 복습 항목 없을 때 "오늘 복습할 내용이 없어요 🎉" 표시
- 우측 하단 **"+ 새 노트"** FAB 버튼
- `useFocusEffect` 사용 — 화면 돌아올 때마다 목록 자동 새로고침

**`app/(tabs)/_layout.tsx` 수정**
- 하단 탭바: 홈 🏠 / 캘린더 📅 / 설정 ⚙️
- `app/(tabs)/calendar.tsx`, `app/(tabs)/settings.tsx` 빈 화면으로 생성

### 트러블슈팅

> ⚠️ `@react-navigation/native` 미설치로 화면 안 뜸
> → `useFocusEffect` import를 `expo-router`로 변경하여 해결

> ⚠️ `@expo/vector-icons` 패키지 내부 파일 누락 오류
> → 아이콘 대신 이모지로 대체하여 해결

> ⚠️ Firestore 복합 쿼리(`userId` + `nextReviewDate`) 사용 시 인덱스 필요 오류
> → `userId`로만 Firestore 검색 후 날짜 필터링을 코드에서 처리하는 방식으로 변경 (인덱스 불필요)
