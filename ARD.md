# ARD — 복습 타이밍 자동 추천 앱

---

## 1. 앱 목적

### 문제 정의
사람은 공부한 내용을 시간이 지날수록 잊어버린다. 문제는 "언제 다시 봐야 하는지"를 스스로 판단하기 어렵다는 것이다. 너무 빨리 복습하면 시간 낭비, 너무 늦게 복습하면 이미 잊어버린 후다.

### 해결책
망각곡선 이론을 기반으로, 사용자가 공부한 내용을 얼마나 잘 기억하는지 테스트한 결과에 따라 **최적의 복습 시점을 자동으로 계산**하고 캘린더에 일정을 잡아준다.
테스트 결과가 좋으면 3일 후, 좋지 않으면 1일 후로 복습 간격을 다르게 설정해 불필요한 반복을 줄이고 효율을 높인다.
또한 스트릭 기능으로 연속 복습 일수를 시각화해 꾸준히 공부하는 습관을 만들 수 있도록 돕는다.

### 타겟 사용자
- 시험을 준비하는 학생
- 자기계발을 위해 꾸준히 공부하는 성인

### 핵심 가치
> 공부한 내용을 언제 다시 봐야 할지 고민하지 않아도 되는 앱

## 2. 주요 기능 목록

### 기능 1 — 노트 작성
- 공부한 내용을 직접 입력하고 노트로 저장
- 과목 또는 태그로 분류 가능

### 기능 2 — 이해도 테스트
두 가지 방식 중 사용자가 선택

- **자가 체크** — 노트 내용을 보고 사용자가 스스로 잘됨 / 안됨 선택
- **AI 테스트** — 입력한 노트 내용을 바탕으로 AI가 문제를 자동 생성, 풀이 결과로 이해도 판단

### 기능 3 — 복습 간격 자동 계산
- 테스트 결과에 따라 다음 복습 날짜 자동 설정
  - 잘됨 → 3일 후 복습
  - 안됨 → 1일 후 복습
- 복습 날짜는 사용자가 직접 수정 가능

### 기능 4 — 복습 캘린더
- 설정된 복습 일정을 캘린더 형태로 한눈에 확인
- 오늘 복습해야 할 항목 목록 표시

### 기능 5 — 스트릭
- 며칠 연속으로 복습했는지 표시
- 하루라도 복습을 빠지면 스트릭 초기화

## 3. 사용자 흐름

### 흐름 1 — 첫 사용 (온보딩)
1. 앱 접속
2. 회원가입 또는 로그인
3. 메인 화면 진입 (오늘의 복습 목록)

### 흐름 2 — 노트 작성
1. 새 노트 작성 버튼 클릭
2. 과목 / 태그 선택
3. 공부한 내용 입력
4. 저장
5. → 첫 복습 일정 자동 생성 (1일 후)

### 흐름 3 — 복습하기
1. 메인 화면에서 오늘 복습할 항목 확인
2. 항목 선택 → 노트 내용 열람
3. 이해도 체크 방식 선택
   - **자가 체크** → 잘됨 / 안됨 직접 선택
   - **AI 테스트** → AI가 생성한 문제 풀기 → 결과로 이해도 자동 판단
4. 결과에 따라 다음 복습 날짜 자동 설정
   - 잘됨 → 3일 후
   - 안됨 → 1일 후
5. 완료 → 스트릭 +1

### 흐름 4 — 캘린더 확인
1. 캘린더 탭 이동
2. 날짜별 복습 일정 한눈에 확인
3. 항목 클릭 → 노트 내용 미리보기

## 4. 화면 목록

### 화면 1 — 로그인 / 회원가입
- **목적:** 사용자 인증
- **UI 요소**
  - 이메일 + 비밀번호 입력 필드
  - 로그인 버튼 / 회원가입 버튼
- **사용자 액션**
  - 로그인 성공 → 메인 화면으로 이동
  - 회원가입 → 정보 입력 후 메인 화면으로 이동

---

### 화면 2 — 메인 화면 (홈)
- **목적:** 오늘 복습해야 할 항목을 한눈에 확인, 스트릭 현황 파악
- **UI 요소**
  - 스트릭 카운터 (연속 복습 일수 표시)
  - 오늘의 복습 목록 (과목명, 노트 제목, 이해도 체크 방식 선택 버튼)
  - 새 노트 작성 버튼
  - 하단 내비게이션 바 (홈 / 캘린더 / 설정)
- **사용자 액션**
  - 복습 항목 클릭 → 노트 상세 화면으로 이동
  - 새 노트 버튼 클릭 → 노트 작성 화면으로 이동

---

### 화면 3 — 노트 작성 화면
- **목적:** 공부한 내용 입력 및 저장
- **UI 요소**
  - 노트 제목 입력 필드
  - 과목 / 태그 선택
  - 본문 내용 입력 필드 (자유 텍스트)
  - 저장 버튼
- **사용자 액션**
  - 저장 → 노트 저장 + 첫 복습 일정 자동 생성 (1일 후) → 메인 화면으로 이동

---

### 화면 4 — 노트 상세 화면
- **목적:** 저장된 노트 내용 열람 및 복습 시작
- **UI 요소**
  - 노트 제목 / 과목 / 태그
  - 본문 내용
  - 다음 복습 날짜 표시
  - 이해도 체크 방식 선택 버튼 (자가 체크 / AI 테스트)
  - 노트 수정 버튼
- **사용자 액션**
  - 자가 체크 선택 → 자가 체크 화면으로 이동
  - AI 테스트 선택 → AI 테스트 화면으로 이동
  - 수정 버튼 → 노트 작성 화면으로 이동 (내용 불러온 상태)

---

### 화면 5 — 자가 체크 화면
- **목적:** 사용자가 스스로 이해도를 판단
- **UI 요소**
  - 노트 본문 내용 표시
  - "잘됨 / 안됨" 선택 버튼
- **사용자 액션**
  - 잘됨 선택 → 3일 후 복습 일정 설정 → 완료 화면
  - 안됨 선택 → 1일 후 복습 일정 설정 → 완료 화면

---

### 화면 6 — AI 테스트 화면
- **목적:** AI가 생성한 문제를 풀고 이해도를 자동으로 판단
- **UI 요소**
  - AI가 생성한 문제 (노트 내용 기반, 1~3문제)
  - 각 문제별 답변 입력 또는 선택지
  - 제출 버튼
  - 결과 요약 (몇 개 맞았는지, 이해도 판정 결과)
- **사용자 액션**
  - 제출 → 결과 분석 → 복습 간격 자동 설정 → 완료 화면

---

### 화면 7 — 캘린더 화면
- **목적:** 전체 복습 일정을 날짜별로 확인
- **UI 요소**
  - 월간 캘린더 뷰
  - 날짜별 복습 항목 수 표시 (점 또는 숫자)
  - 날짜 선택 시 해당일 복습 목록 표시
- **사용자 액션**
  - 날짜 클릭 → 해당일 복습 목록 표시
  - 항목 클릭 → 노트 상세 화면으로 이동

---

### 화면 8 — 설정 화면
- **목적:** 앱 환경설정
- **UI 요소**
  - 계정 정보 (이메일, 로그아웃)
  - 복습 알림 시간 설정
  - 기본 복습 간격 설정 (잘됨 / 안됨 기본값 수정 가능)
- **사용자 액션**
  - 각 항목 수정 → 저장
