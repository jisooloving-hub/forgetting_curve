⚛️
React Native + Expo

Firebase

react-native-calendars

07 APP STRUCTURE

앱 구조 — 디렉토리

레이어별 역할 분리

forgetting-curve/ ├── app/ # Expo Router 화면 │ ├── (auth)/ │ │ ├── login.tsx │ │ ├── signup.tsx │ │ └── forgot-password.tsx │ ├── (tabs)/ │ │ ├── index.tsx # 홈 │ │ ├── calendar.tsx │ │ └── settings.tsx │ └── note/ │ ├── create.tsx │ ├── list.tsx │ ├── [id].tsx # 동적 라우트 │ └── quiz.tsx ├── services/ # 비즈니스 로직 │ ├── firebase.ts │ ├── auth.ts │ └── notes.ts ├── types/index.ts # TypeScript 타입 └── app.json # Expo 설정
🖥 Presentation Layer (app/)
화면 렌더링과 사용자 인터랙션 처리. Expo Router로 파일 경로 = URL 경로. 인증/탭/노트 3개 영역으로 분리.
⚙️ Service Layer (services/)
Firebase 직접 호출 로직 집중. 화면 컴포넌트는 서비스 함수만 호출 → 관심사 분리로 유지보수성 향상.
☁️ Cloud Layer (Firebase)
Auth로 사용자 인증 관리, Firestore로 notes 컬렉션 저장. 실시간 DB 대신 getDoc/getDocs로 필요 시 조회.
💾 Local Cache (AsyncStorage)
스트릭 날짜·횟수, NEW 뱃지 기준 타임스탬프 저장. Firebase 호출 없이 빠른 로컬 상태 유지.
08 HOW WE BUILT IT

구현 방법

주요 기능별 구현 방식

⏰
망각곡선 복습일 계산
  • 노트 최초 작성 시 nextReviewDate = today + 1
  • 퀴즈 결과에 따라 복습일 조정
    잘됐어요 +3일 / 보통 +2일 / 헷갈려요 +1일
  • 반복할수록 간격이 늘어나는 간격 반복 학습 구현
🤖
AI 퀴즈 생성 (자체 엔진)
  • 노트 내용을 문장 단위로 분리
  • 각 문장에서 가장 긴 단어 = 핵심어 추출
  • /다$/ 정규식으로 동사/명사 구분
  • 같은 품사의 단어를 오답으로 사용 → 자연스러운 선지
  • 최대 5문제 동적 생성 (문장 수 따라 달라짐)
🔥
스트릭 (연속 복습일)
  • 퀴즈 완료 시 AsyncStorage에 날짜 + 횟수 저장
  • 어제 날짜면 +1, 아니면 1로 초기화
  • 오늘 이미 완료했으면 중복 카운트 방지
  • 홈 화면 포커스 시 AsyncStorage에서 불러와 표시
📅
캘린더 복습 시각화
  • react-native-calendars multi-dot 마킹 방식
  • 노트의 과목 색상 = 점 색상으로 시각 구분
  • 날짜 클릭 시 해당 날 복습 예정 노트 목록 표시
  • Expo useFocusEffect로 화면 재진입 시 자동 갱신
09 CHALLENGES

구현 시행착오

개발 과정에서 만난 문제와 해결 방법

01
🚫 Anthropic API CORS 오류 — 웹 브라우저에서 API 직접 호출 불가
원인: 브라우저는 외부 API에 직접 요청 불가 (보안 정책). 해결: Node.js로 로컬 CORS 프록시(proxy.js) 작성, 웹에서는 프록시 경유, 모바일에서는 직접 호출 분기 처리.
해결 ✓
02
🚫 Gemini API 할당량 0 — 학교 이메일 계정 사용 제한, 개인 Gmail도 동일
원인: 학교 조직 계정의 Google Workspace 정책 + 한국 지역 무료 할당량 제한. 해결: 외부 API 의존 제거, 자체 퀴즈 생성 엔진 구현 → 오프라인에서도 동작, API 비용 0원.
해결 ✓
03
🚫 퀴즈 오답 선지 품질 불량 — 동사 답에 명사 오답, 또는 선지 4개가 모두 동일
원인: 오답 풀이 단어 수 부족 + 품사 미구분. 해결: 정규식 /다$/ 으로 동사/명사 판별, 동일 품사 단어만 오답으로 사용, 부족 시 사전 정의 VERB_FALLBACKS/NOUN_FALLBACKS에서 보충.
해결 ✓
04
🚫 퀴즈 진행 오류 — 3문제짜리 노트인데 4번 문제에서 멈춤
원인: handleNext가 currentQ < 4로 하드코딩되어 있어 실제 문제 수와 불일치. 해결: currentQ < questions.length - 1로 동적 처리.
해결 ✓

14 ADR

Architecture Decision Records

주요 기술 결정 사항 — 질의응답 대비

ADR-001 React Native + Expo 채택 Accepted
CONTEXT
iOS·Android·Web 모두 지원하는 앱이 필요했고, 개발 인원이 1명이었음
DECISION
React Native + Expo SDK 56 채택. Expo Router로 파일 기반 라우팅 적용
CONSEQUENCE
단일 코드베이스로 3개 플랫폼 지원. 네이티브 모듈 접근은 Expo Modules API로 제한
ADR-002 Firebase (BaaS) 선택 — 자체 서버 대신 Accepted
CONTEXT
백엔드 서버를 직접 구축할 시간이 없었고, 인증·DB를 빠르게 붙여야 했음
DECISION
Firebase Auth(이메일 인증) + Firestore(노트 저장)로 서버리스 구현
CONSEQUENCE
서버 구축·운영 비용 0원. 단, Firestore 복잡한 쿼리는 클라이언트 필터링으로 처리
ADR-003 AI 퀴즈: 외부 API → 자체 엔진으로 전환 Accepted
CONTEXT
Anthropic API 결제 오류, Gemini API 한국 학교 계정 할당량 0 — 두 API 모두 사용 불가 상황 발생
DECISION
문장 파싱 + 핵심어 추출 + 품사 분류 기반 자체 퀴즈 생성 엔진 구현
CONSEQUENCE
API 비용 0원, 오프라인 동작 가능. 단, 복잡한 추론형 문제는 생성 불가

🎯 자기 점검
단순 읽기가 아닌 퀴즈 풀기로 능동적 회상(Active Recall) 효과. 시험 적중률 향상.
